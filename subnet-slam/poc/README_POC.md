# PoC: видео + промпт → метрики (OpenAI)

Скрипт: `video_validator_openai_poc.py` (не трогает основной код subnet).

## Как анализировать видео с ChatGPT / OpenAI

**Из коробки в Chat Completions нет «загрузить MP4 одним файлом» для `gpt-4o-mini`.** Мультимодальность идёт через **картинки** (`image_url`: URL или `data:image/jpeg;base64,...`). Поэтому для PoC стандартный путь:

1. **Равномерно выбрать N кадров** из ролика (у нас OpenCV).
2. **Сжать в JPEG** с умеренным качеством, `detail: low` — меньше токенов на изображение.
3. **Один запрос** с текстом задачи + подписи «кадр # / секунда» + картинки.

Альтернативы (не в этом скрипте):

- **FFmpeg** нарезать кадры на диск — то же самое, больше файлов.
- **Другие провайдеры** (например, Gemini) иногда принимают длинное видео одним входом — другая цена и API.
- **Assistants + file_search / code interpreter** — другой продуктовый контур, обычно не то, что нужно для дешёвого онлайн-скоринга в валидаторе.

## Дешёвые настройки

- Модель: **`gpt-4o-mini`** (vision, дешевле `gpt-4o`).
- Мало кадров: **`--max-frames 8`** … **12** — компромисс цена / охват длины эпизода.
- JPEG пониже, в коде уже `detail: low`.

## Запуск

```bash
cd subnet-vla/poc
pip install -r requirements-poc.txt
export OPENAI_API_KEY=sk-...
python video_validator_openai_poc.py --video /path/to/clip.mp4 --prompt "Pick the red block and place it in the bin." --max-frames 10
```

Вывод: JSON (Pydantic) в stdout.

## Ограничения PoC

- Нет проверки на «одни и те же» кадры при коротком ролике — только равномерная сетка.
- Оценка субъективна; для продакшена нужны рубрики, калибровка и (часто) человек в петле.
